HTML Comments

Komentar HTML adalah teks di dalam kode yang sengaja disembunyikan dari browser. Teks ini tidak akan terlihat oleh pengunjung website Anda.

Sintaks Penulisan:

Dimulai dengan <!-- dan diakhiri dengan -->.

  • Ada tanda seru (!) di awal, tapi tidak ada di akhir.
  • Isi di dalamnya bebas dan tidak diproses sebagai kode.

Contoh Kode:

<!-- Ini adalah komentar yang tidak akan muncul di layar -->
<p>Paragraf ini akan muncul di layar.</p>

Peringatan Keamanan!

Meskipun tidak tampil di layar browser, siapapun bisa melihat komentar Anda melalui fitur "View Page Source" (Klik kanan di browser). Jangan pernah menaruh informasi rahasia seperti password atau data pribadi di dalam komentar HTML!

Mengapa Kita Butuh Komentar?

Seiring bertambahnya baris kode Anda, membaca file HTML akan menjadi semakin sulit. Komentar membantu memberikan navigasi.

Kegunaan Utama:
  • Dokumentasi: Menjelaskan fungsi bagian kode tertentu.
  • Organisasi: Menandai awal dan akhir sebuah bagian (misal: header, footer, sidebar).
  • Catatan Tim: Memberitahu rekan kerja tentang perubahan yang dilakukan.

Contoh Organisasi Kode:

<!-- BAGIAN NAVIGASI START -->
<nav>
    <ul>
        <li>Home</li>
        <li>About</li>
    </ul>
</nav>
<!-- BAGIAN NAVIGASI END -->

Tips Pro: Shortcut Keyboard

Di hampir semua Code Editor (seperti VS Code), Anda bisa membuat komentar secara instan dengan menyorot teks lalu menekan Ctrl + / (Windows) atau Cmd + / (Mac).

Debugging & Testing

Komentar sering digunakan untuk menyembunyikan sementara bagian kode yang bermasalah tanpa harus menghapusnya.

Istilah Baru: Debugging

Proses mencari dan memperbaiki kesalahan (bug) di dalam kode program.

Cara Menyembunyikan Kode:

Jika Anda punya gambar yang rusak atau teks yang belum pasti, "bungkus" dengan komentar:

<h1>Judul Berita</h1>
<!-- 
<img src="foto-rusak.jpg" alt="Foto Berita"> 
<p>Teks berita yang belum selesai ditulis...</p>
-->

Kasus Nyata: Eksperimen Desain

Bayangkan Anda punya dua pilihan desain tombol. Daripada menghapus salah satu, Anda bisa memberikan komentar pada tombol pertama untuk melihat hasil tombol kedua. Jika hasilnya buruk, Anda tinggal memindahkan komentarnya kembali.
