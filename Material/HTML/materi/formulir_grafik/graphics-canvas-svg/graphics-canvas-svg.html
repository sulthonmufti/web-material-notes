Canvas vs SVG

Keduanya digunakan untuk membuat grafis di web, tetapi cara kerja dan tujuannya sangat berbeda.

Fitur SVG Canvas
Format Vektor (XML) Raster (Pixel)
Skalabilitas Tidak pecah saat dizoom Pecah jika diperbesar
Kontrol Via CSS & DOM Via JavaScript

Kapan Pakai yang Mana?

Gunakan SVG untuk logo, ikon, dan diagram sederhana. Gunakan Canvas untuk game 2D/3D, pemrosesan video, atau animasi kompleks dengan ribuan objek.

Scalable Vector Graphics (SVG)

SVG mendefinisikan grafis dalam format XML. Artinya, setiap elemen (lingkaran, kotak, garis) adalah tag HTML yang bisa Anda manipulasi.

Tag SVG Dasar:
  • <svg>: Wadah utama.
  • <circle>: Membuat lingkaran (cx, cy, r).
  • <rect>: Membuat kotak (width, height).
  • <path>: Membuat bentuk bebas yang kompleks.

Contoh Kode SVG:

<svg width="100" height="100">
  <circle cx="50" cy="50" r="40" stroke="purple" stroke-width="4" fill="yellow" />
</svg>

Lingkaran di atas digambar langsung dengan kode HTML!

HTML5 Canvas

Canvas hanyalah sebuah "kanvas kosong". Anda tidak bisa menggambar apa pun di dalamnya tanpa bantuan JavaScript.

Istilah Baru: Rendering Context

Objek dalam JavaScript yang menyediakan metode untuk menggambar. Versi yang paling umum adalah 2d.

Langkah Menggunakan Canvas:

  1. Buat elemen <canvas> di HTML.
  2. Ambil elemen tersebut di JavaScript.
  3. Gambar menggunakan fungsi seperti fillRect() atau arc().
<canvas id="myCanvas" width="200" height="100"></canvas>

<script>
  var c = document.getElementById("myCanvas");
  var ctx = c.getContext("2d");
  ctx.fillStyle = "#9b59b6";
  ctx.fillRect(20, 20, 150, 60);
</script>

Kasus Nyata: Dashboard & Game

Jika Anda melihat grafik saham yang bergerak naik turun secara real-time atau game seperti Flappy Bird di web, itu hampir pasti dibuat menggunakan elemen <canvas>.
