Kontrol Akses Input

Bagaimana kita mengatur data awal dan apakah user boleh mengubah data tersebut?

Istilah Penting:
  • value: Menentukan nilai awal (default) yang ada di dalam input.
  • readonly: User bisa melihat dan klik, tapi tidak bisa menghapus/mengubah teks. Data tetap terkirim ke server.
  • disabled: Input "mati". Tidak bisa diklik dan datanya **tidak akan** terkirim ke server.

Contoh Kode:

<input type="text" value="ID-12345" readonly>
<input type="text" value="Tidak Aktif" disabled>

Membatasi Karakter

Penting untuk validasi agar database Anda tidak menerima teks yang terlalu panjang atau tidak sesuai format.

Atribut Pembatas:
  • maxlength: Jumlah karakter maksimal yang boleh diketik.
  • minlength: Jumlah karakter minimal agar form dianggap valid.
  • size: Mengatur lebar fisik kotak input (dalam jumlah karakter).

Contoh Kasus: Kode Pos

Kode pos di Indonesia selalu 5 angka. Kita bisa membatasi agar user tidak mengetik lebih dari itu.

<input type="text" name="kodepos" maxlength="5" size="5">

Atribut Angka (Min, Max, Step)

Khusus untuk type="number" dan type="range", kita bisa mengatur rentang nilai yang diizinkan.

Atribut Numerik:
  • min: Nilai terkecil yang boleh dimasukkan.
  • max: Nilai terbesar yang boleh dimasukkan.
  • step: Kelipatan angka (misal: hanya boleh kelipatan 10).
  • multiple: (Khusus file/email) Mengizinkan memilih lebih dari satu data.

Contoh Kode:

<!-- Input angka 10-100 kelipatan 10 -->
<input type="number" min="10" max="100" step="10">

Kasus Nyata: Pembelian Produk

Dalam toko online, atribut min="1" memastikan pembeli tidak memesan 0 barang, dan max bisa disesuaikan dengan sisa stok yang ada di gudang.
