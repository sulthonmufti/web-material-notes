Struktur Dasar Form

Form HTML dibungkus menggunakan tag <form>. Tag ini berfungsi sebagai pengumpul data sebelum dikirimkan ke server.

Atribut Utama Form:
  • action: URL tujuan ke mana data akan dikirim (biasanya file PHP atau server).
  • target: Menentukan apakah hasil kiriman dibuka di tab baru (_blank) atau tab yang sama.

Contoh Kode:

<form action="/proses-data.php">
  <label for="nama">Nama:</label>
  <input type="text" id="nama" name="user_nama">
  <input type="submit" value="Kirim">
</form>

Kasus Nyata: Form Pencarian

Pada kotak pencarian Google, saat Anda menekan Enter, atribut action mengarahkan data teks Anda ke halaman hasil pencarian mereka.

Method: GET vs POST

Bagaimana cara data dikirimkan? Ada dua cara utama yang sering digunakan melalui atribut method.

Perbedaan Utama:
  • GET: Data ditempelkan langsung di URL (terlihat oleh mata). Cocok untuk pencarian (search).
  • POST: Data dikirim "di balik layar". Sangat penting untuk data sensitif seperti password.

Kapan Pakai yang Mana?

Gunakan POST untuk pendaftaran akun atau kirim file. Gunakan GET untuk fitur filter atau pencarian produk yang bisa di-bookmark URL-nya oleh pengguna.

Atribut Input Wajib

Tanpa atribut yang benar, server tidak akan mengenali data apa yang Anda kirimkan.

Istilah Baru:
  • name: Nama variabel yang akan dikenali server (Wajib ada!).
  • value: Nilai default atau nilai yang akan dikirim.
  • placeholder: Teks bantuan sementara di dalam kotak input.
  • required: Memaksa pengguna mengisi kotak tersebut sebelum dikirim.
