1. Pengenalan Function +

Function adalah blok kode yang dirancang untuk melakukan tugas tertentu. Function baru akan berjalan ketika "dipanggil" (invoked).

üìñ Istilah Baru: Declaration & Call
  • Declaration: Mendefinisikan/membuat function.
  • Call/Invoke: Menjalankan/memanggil function tersebut.
function sapaUser() {
    console.log("Halo, Selamat Belajar!");
}

// Memanggil function
sapaUser();
2. Parameters & Arguments +

Agar function lebih fleksibel, kita bisa mengirimkan data ke dalamnya.

üìñ Istilah: Parameter vs Argument
  • Parameter: Variabel yang ditulis saat membuat function (seperti "lubang input").
  • Argument: Nilai asli yang dimasukkan saat memanggil function.

üõ†Ô∏è Dapur Rekaman: Bedah Input

function greet(nama) { // 'nama' adalah PARAMETER
    return "Halo " + nama;
}

greet("Budi"); // "Budi" adalah ARGUMEN

Proses: Saat panggilSapaan() dijalankan, ia mengambil teks dari input dan mengirimnya sebagai "bahan baku" ke dalam parameter function.

3. Return Keyword +

return digunakan untuk mengeluarkan hasil dari function agar bisa disimpan ke dalam variabel lain atau digunakan di tempat lain.

‚ö†Ô∏è Penting: Batas Akhir Setelah return dijalankan, function akan berhenti. Kode apapun di bawah return tidak akan pernah dieksekusi.

Hitung Luas Persegi (Sisi x Sisi):

üõ†Ô∏è Dapur Rekaman: Kenapa butuh Return?

function hitungLuas(s) {
    return s * s;
}

let hasil = hitungLuas(5); // Nilai 25 sekarang "pulang" dan disimpan di variabel 'hasil'
4. Arrow Functions (ES6) +

Cara modern dan lebih singkat untuk menulis function. Sangat sering digunakan dalam pengembangan Frontend saat ini (seperti di React).

üìñ Istilah: Fat Arrow (=>) Menghilangkan kata kunci function dan menggantinya dengan tanda panah. 
// Cara Biasa
function (a, b) { return a + b; }

// Cara Arrow Function
const tambah = (a, b) => a + b;

üõ†Ô∏è Dapur Rekaman: Kelebihan Arrow Function

  • Lebih Singkat: Jika hanya satu baris, tidak perlu kurung kurawal {} dan tidak perlu kata return.
  • this Keyword: Arrow function tidak memiliki this sendiri (sangat berguna saat menangani event di JavaScript lanjut).
