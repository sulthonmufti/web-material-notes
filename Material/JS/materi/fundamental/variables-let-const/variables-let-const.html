JavaScript: Variables

Menyimpan Data di Memori Browser

1. Apa itu Variabel? +

Variabel adalah "wadah" untuk menyimpan nilai data. Bayangkan variabel sebagai kotak berlabel di gudang (memori komputer) tempat Anda menaruh barang agar bisa diambil lagi nanti.

üìñ Istilah: Declaration & Assignment
  • Declaration: Membuat kotak (Contoh: let nama;)
  • Assignment: Mengisi kotak dengan barang (Contoh: nama = "Budi";)
2. Let vs Const (Standard Modern) +

Di JavaScript modern (ES6), kita hampir selalu menggunakan let atau const.

Keyword Kegunaan Bisa Diubah nilainya?
let Data yang nilainya bisa berubah-ubah. Ya
const Data tetap (Konstanta). Tidak

Kasus Penggunaan:

const pi = 3.14; // Nilai matematika tidak akan berubah
let skor = 0;   // Skor game akan terus bertambah
skor = 10;      // Berhasil diubah!
3. Kenapa 'var' Tidak Lagi Dipakai? +

Dulu hanya ada var. Namun, var punya masalah dengan "Scope" (ruang lingkup).

üìñ Istilah: Scope Scope menentukan di mana variabel bisa diakses. let dan const menggunakan Block Scope (hanya hidup di dalam kurung kurawal {}).

Bahaya Var: var tidak peduli dengan block scope, sehingga sering menyebabkan bug "kebocoran" variabel yang tidak disengaja.

1. Kenapa 'Let' Berhasil Berubah?

let skor = 50; 
        skor = 100; // Browser mengizinkan penimpaan nilai di alamat memori yang sama.

Proses: Saat let dideklarasikan, JavaScript menyiapkan slot memori yang "terbuka". Ketika baris kedua dijalankan, nilai 50 dibuang dan diganti dengan 100.

2. Kenapa 'Const' Menghasilkan Error?

const nama = "Gemini";
try {
    nama = "AI"; // Browser memblokir perubahan pada variabel bertipe konstanta.
} catch (err) {
    console.error(err.message); // Menangkap pesan "Assignment to constant variable"
}

Proses:

  • Try...Catch: Ini adalah "jaring pengaman". Karena mengubah const akan membuat program crash (berhenti mendadak), kita bungkus di dalam try.
  • Jika terjadi kesalahan, alur dipindahkan ke blok catch, sehingga kita bisa menampilkan pesan error yang ramah kepada user tanpa menghentikan seluruh website.
