Git Init

Mengubah Folder Biasa Menjadi Repositori Cerdas

1. Apa itu git-init? +

git init adalah perintah untuk membuat repositori Git baru. Ini adalah langkah pertama yang Anda lakukan saat memulai proyek baru secara lokal.

ğŸ“– Penjelasan:

Perintah ini menciptakan folder tersembunyi bernama .git. Di dalam folder inilah Git menyimpan semua sejarah, snapshot, dan metadata proyek Anda. Tanpa folder ini, Git tidak bisa melacak perubahan apa pun.

ğŸ¢ Kasus Real: Inisialisasi Proyek Baru

Bayangkan Anda baru saja membuat folder toko-online-v1. Untuk mulai merekam setiap perubahan kode, Anda wajib masuk ke folder tersebut lewat terminal dan mengetik git init.

2. Menjalankan Perintah +

Ada dua cara umum untuk menggunakan perintah ini:

# Cara 1: Di dalam folder proyek yang sudah ada
cd nama-proyek
git init

# Cara 2: Membuat folder sekaligus inisialisasi
git init nama-folder-baru

3. Efek Tersembunyi +

Setelah Anda menjalankan perintah tersebut, Git akan membalas dengan pesan: "Initialized empty Git repository in..."

ğŸ•µï¸ Folder .git:

Folder ini bersifat tersembunyi (hidden). Di Windows atau Mac, Anda mungkin tidak melihatnya secara langsung. Gunakan perintah ls -a (Linux/Mac) atau dir /a (Windows) untuk melihatnya.

Peringatan: Jangan pernah menghapus folder .git kecuali Anda ingin membuang seluruh riwayat versi proyek Anda dan menjadikannya folder biasa lagi.

4. Cabang Utama (Default Branch) +

Secara tradisional, Git akan membuat branch pertama bernama master. Namun, standar industri modern kini beralih menggunakan nama main.

ğŸ¢ Standar Industri Modern

Anda bisa mengatur agar Git selalu menggunakan nama main setiap kali inisialisasi dengan perintah config:

git config --global init.defaultBranch main
