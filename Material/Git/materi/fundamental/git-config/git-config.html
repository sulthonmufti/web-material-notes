Git Config

Mengatur Identitas dan Personalisasi Lingkungan Kerja Git

1. Apa itu git-config? +

git config adalah perintah yang digunakan untuk mengatur bagaimana Git berperilaku di komputer Anda. Ini mencakup segala hal, mulai dari siapa Anda, hingga editor apa yang Anda gunakan untuk menulis pesan commit.

2. Tingkatan Konfigurasi (Levels) +

Git memiliki tiga tempat penyimpanan konfigurasi yang berbeda. Aturan mainnya: Konfigurasi yang paling spesifik akan menang.

Level Cakupan (Scope) Flag Perintah
System Seluruh pengguna di OS komputer ini. --system
Global Seluruh proyek milik user Anda saat ini. --global
Local Hanya untuk satu folder proyek tertentu. --local
ğŸ¢ Case Industri: Kantor vs Pribadi

Anda mungkin menggunakan email kantor secara Global, tetapi untuk proyek *side-job* pribadi di folder tertentu, Anda menggunakan Local config agar kontribusinya tercatat ke email pribadi Anda.

3. Identitas & Editor +

Berikut adalah pengaturan "Starter Pack" yang harus dilakukan setiap kali menginstal Git baru:

# 1. Mengatur Nama
git config --global user.name "Budi Santoso"

# 2. Mengatur Email (Sesuai akun GitHub/GitLab)
git config --global user.email "budi@email.com"

# 3. Mengatur VS Code sebagai Default Editor
git config --global core.editor "code --wait"

4. Cara Cek & Verifikasi +

Jangan sampai salah kirim email! Selalu cek konfigurasi Anda sebelum mulai bekerja.

# Melihat semua konfigurasi yang aktif
git config --list

# Melihat asal file setiap konfigurasi
git config --list --show-origin

# Hanya cek satu item spesifik
git config user.email
