Git Add

Menyiapkan File di Staging Area Sebelum Menjadi Sejarah

1. Apa itu git-add & Staging Area? +

git add adalah perintah untuk memindahkan perubahan dari Working Directory ke Staging Area (sering disebut juga Index).

ğŸ“– Istilah: Staging Area

Bayangkan ini sebagai "ruang tunggu" atau "ruang ganti". Di sini Anda mengumpulkan semua file yang sudah siap untuk dibungkus menjadi satu paket (Commit). File yang sudah di-add statusnya menjadi Staged.

ğŸ¢ Kasus Real: Kurasi Perubahan

Bayangkan Anda memperbaiki 3 bug, tapi bug ke-3 belum selesai tes. Anda menggunakan git add hanya pada file bug 1 dan 2. Dengan begitu, Anda tetap bisa melakukan commit yang "bersih" tanpa menyertakan kode yang masih setengah matang.

2. Cara Menggunakan git-add +

Ada beberapa cara untuk menambahkan file, tergantung seberapa banyak perubahan yang ingin Anda masukkan:

# Menambah satu file spesifik
git add index.html

# Menambah semua file yang berubah di folder saat ini
git add .

# Menambah file berdasarkan pola (misal: semua file CSS)
git add *.css

# Mode interaktif (memilih bagian kode di dalam file)
git add -p

3. Memantau Perubahan dengan git-status +

Setiap kali Anda menjalankan git add, Anda wajib mengecek hasilnya menggunakan git status.

âœ… Tips: Perhatikan Warnanya
  • Merah: File dimodifikasi tapi belum di-add (Changes not staged).
  • Hijau: File sudah di-add dan siap di-commit (Changes to be committed).
4. Pembatalan Staging (Unstaging) +

Bagaimana jika Anda tidak sengaja melakukan git add . dan ingin membatalkan satu file agar tidak ikut di-commit?

Gunakan perintah reset:

git restore --staged nama-file.js
