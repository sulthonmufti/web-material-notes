Git Architecture

Menjelajahi Struktur Objek dan Folder Rahasia .git

1. Content-Addressable Storage +

Arsitektur utama Git adalah Content-Addressable Storage. Artinya, Git tidak mencari file berdasarkan nama (misal: index.html), melainkan berdasarkan isinya.

Jika Anda punya 10 file berbeda dengan isi yang 100% sama, Git hanya akan menyimpan satu objek data. Ini membuat ukuran penyimpanan sangat efisien.

ğŸ¢ Case Industri: Duplikasi Aset

Dalam proyek web besar, seringkali ada gambar atau file konfigurasi yang sama di folder berbeda. Karena arsitektur ini, Git tidak akan membengkak ukurannya karena ia tahu isinya sama.

2. The Big Four: Blob, Tree, Commit, Tag +

Git menyimpan semuanya dalam 4 tipe objek di folder .git/objects:

  • Blob (Binary Large Object): Hanya menyimpan isi file (tanpa nama file).
  • Tree: Seperti folder. Ia memetakan nama file ke Blob dan menyimpan struktur sub-folder.
  • Commit: Berisi link ke Tree utama, nama penulis, pesan commit, dan link ke commit sebelumnya (parent).
  • Tag: Nama alias yang statis untuk sebuah commit (misal: v1.0).

ğŸ” Contoh Visual di Terminal:

Kita bisa melihat isi objek menggunakan perintah "low-level" (Plumbing):

# Melihat tipe objek dari sebuah Hash
git cat-file -t 1a2b3c

# Melihat isi objek
git cat-file -p 1a2b3c
3. Anatomi Folder .git +

Inilah organ dalam Git yang ada di proyek Anda:

Folder/File Fungsi
config Pengaturan spesifik proyek ini saja.
HEAD File yang menunjuk ke branch yang aktif saat ini.
index Tempat Staging Area (berupa file biner).
objects/ Database semua Blob, Tree, dan Commit Anda.
refs/ Penyimpan pointer ke branch dan tag.
4. Bagaimana Branch Bekerja? +

Berbeda dengan sistem lain yang meng-copy seluruh folder, di Git, **Branch hanyalah sebuah file teks kecil**.

ğŸ“– Konsep Penting:

Branch main hanyalah file teks yang isinya adalah 40 karakter Hash dari commit terakhir. Saat Anda commit baru, isi file teks ini diperbarui ke Hash yang baru.

Kenapa Git "Ringan"?

Membuat 1000 branch di Git hanya memakan beberapa Kilobyte memori karena Git hanya membuat 1000 file teks kecil yang menunjuk ke commit.

ğŸ¢ Case Industri: "Detached HEAD"

Anda mungkin pernah mengalami error ini. Secara arsitektur, ini artinya file HEAD Anda menunjuk langsung ke sebuah Commit Hash, bukan ke sebuah Branch. Git memperingatkan Anda bahwa jika Anda commit, tidak ada branch yang akan mencatatnya.
