CSS Specificity & !important

Apa itu Specificity?

Jika ada dua atau lebih aturan CSS yang bertabrakan menargetkan elemen yang sama, browser akan memilih aturan yang paling **spesifik**.

Analogi: Jika seseorang memanggil "Halo Orang!" (General), dan orang lain memanggil "Halo Budi!" (Spesifik), maka Budi akan menoleh ke panggilan kedua.

Contoh Kasus:

p { color: green; }
.teks-merah { color: red; }

Jika elemennya adalah <p class="teks-merah">Teks ini</p>, maka warnanya akan MERAH karena Class lebih spesifik daripada Tag elemen.

Hierarki Poin (Cara Menghitung)

Browser menggunakan sistem poin untuk menentukan siapa yang menang:

  1. Inline Styles (1-0-0-0): Menang terhadap semua CSS di file eksternal.
  2. IDs (0-1-0-0): Sangat kuat. Menang melawan class.
  3. Classes, Attributes, Pseudo-classes (0-0-1-0): Seperti .menu atau :hover.
  4. Elements & Pseudo-elements (0-0-0-1): Seperti h1, p, div.

Teks ini akan tetap BIRU (ID menang melawan Class dan Tag)

Aturan Tambahan: Jika poinnya sama persis (misal dua class yang berbeda warna), maka aturan yang ditulis **paling bawah (terakhir)** di file CSS yang akan menang.

Penggunaan !important

Kata kunci !important digunakan untuk memaksa suatu aturan CSS agar selalu menang, mengabaikan semua aturan Specificity.

.kotak-kuning { background-color: yellow; }
.kotak-hijau { background-color: green !important; }
Meskipun kuning ditulis lebih dulu atau ada aturan lain, kotak ini akan tetap HIJAU karena !important.
⚠️ PERINGATAN: Hindari menggunakan !important kecuali benar-benar terpaksa (misal saat memperbaiki bug pihak ketiga). Terlalu banyak !important akan membuat CSS Anda berantakan dan sangat sulit untuk didebug di masa depan.
