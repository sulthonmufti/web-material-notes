Navigation Bars & Dropdowns

1. Horizontal Navbar (Flexbox)

Cara modern membuat Navbar adalah menggunakan elemen <nav>, <ul>, dan Flexbox untuk mengatur posisinya.

Analogi Struktur:

  • Container (nav): Ruang utama/jalan raya.
  • List (ul): Kereta belanja yang membawa item.
  • Links (a): Tombol interaktif yang bisa diklik.
.navbar {
  display: flex;
  justify-content: space-between; /* Logo di kiri, Menu di kanan */
  align-items: center;
}
.nav-links {
  display: flex; /* Membuat menu berjejer ke samping */
  list-style: none;
}

2. Membuat Dropdown Menu

Dropdown bekerja dengan prinsip "Sembunyi dan Muncul". Kita menggunakan CSS untuk mendeteksi kapan kursor berada di atas menu induk, lalu mengubah status tampilan anak elemennya.

A. Struktur HTML yang Benar

Menu dropdown harus diletakkan di dalam elemen daftar (<li>) yang menjadi induknya. Ini penting agar saat kursor mengenai dropdown, kursor dianggap masih berada di dalam area induk.

<li class="dropdown">
  <a href="#">Menu Induk</a>
  <!-- Menu yang akan disembunyikan -->
  <div class="dropdown-content">
    <a href="#">Sub Menu 1</a>
    <a href="#">Sub Menu 2</a>
  </div>
</li>

B. Bedah Kode CSS (Logika Utama)

Ada tiga kunci utama agar dropdown ini berfungsi:

  1. Positioning: Induk diberi relative, dan isi dropdown diberi absolute.
  2. Initial State: Isi dropdown disembunyikan dengan display: none.
  3. Hover State: Menggunakan selector .induk:hover .anak untuk memunculkan kembali.
/* 1. Sembunyikan menu */
.dropdown-content {
  display: none;
  position: absolute;
  min-width: 160px;
  box-shadow: 0px 8px 16px rgba(0,0,0,0.2);
  z-index: 1; /* Pastikan berada di atas konten lain */
}

/* 2. Tampilkan saat Mouse mendekat */
.dropdown:hover .dropdown-content {
  display: block;
}

C. Kenapa Harus Position Relative & Absolute?

Analogi: position: relative pada induk adalah seperti "jangkar". Tanpa jangkar ini, menu dropdown (yang memiliki position: absolute) akan melompat keluar dan menempel pada pojok layar browser, bukan tepat di bawah tombol menunya.

3. Sticky & Fixed Navbar

Kedua properti ini digunakan untuk menjaga agar navigasi tetap terlihat saat user melakukan scroll, namun cara kerjanya berbeda secara teknis.

(Scroll area kotak ini ke bawah untuk melihat perbedaan)

Fixed: Menempel di Layar Utama
Area Konten Awal
Sticky: Menempel saat mencapai atas

Konten panjang...

Kotak Hijau (Sticky) akan menempel di atas area scroll ini.

Kotak Merah (Fixed) akan terus menempel di layar Anda tidak peduli apapun yang terjadi, karena ia keluar dari aliran dokumen.

A. Position: Fixed

  • Karakter: Elemen "dicabut" dari aliran HTML normal.
  • Efek: Elemen di bawahnya akan naik ke atas seolah-olah elemen Fixed tersebut tidak ada (seringkali menabrak/tertutup).
  • Kegunaan: Cocok untuk tombol "Back to Top" atau Chat Widget.
.navbar-fixed {
  position: fixed;
  top: 0;
  width: 100%;
}

B. Position: Sticky

  • Karakter: Elemen tetap berada di posisi aslinya sampai layar di-scroll menyentuh angka top yang ditentukan.
  • Efek: Elemen di bawahnya tidak akan naik. Layout tetap rapi.
  • Kegunaan: Sangat populer untuk Navbar utama website modern.
.navbar-sticky {
    position: sticky;
    top: 0; /* Menempel saat jarak ke atas layar 0px */
    }
Peringatan Sticky: Agar sticky bekerja, elemen induknya (parent) tidak boleh memiliki properti overflow: hidden. Selain itu, sticky hanya akan menempel selama ia masih berada di dalam area parent-nya.

Apa itu Z-Index?

Saat menggunakan fixed atau sticky, Anda wajib menambahkan z-index: 1000;. Ini memastikan navbar Anda berada di lapisan paling depan dan tidak "tenggelam" di bawah gambar atau teks lain saat di-scroll.
