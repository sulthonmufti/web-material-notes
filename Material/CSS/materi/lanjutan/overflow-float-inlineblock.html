Overflow, Float, & Inline-block

1. Overflow

Overflow mengatur apa yang terjadi jika konten di dalam sebuah elemen terlalu besar untuk ukuran kotaknya (wadahnya).

Nilai yang Sering Digunakan:

  • visible (Default): Konten tetap muncul keluar kotak.
  • hidden: Konten yang meluap akan dipotong dan disembunyikan.
  • scroll: Selalu memunculkan scrollbar (meskipun konten tidak meluap).
  • auto: Memunculkan scrollbar hanya jika konten meluap.

Kasus Nyata: Kotak Chat atau Berita

Gunakan overflow: auto untuk area yang terbatas tapi memiliki banyak teks.

Ini adalah teks yang sangat panjang. Dengan overflow: auto, kotak ini akan memiliki scrollbar sendiri sehingga tidak merusak tata letak elemen lain di sekitarnya. Scroll ke bawah untuk membaca lebih lanjut! 
.container {
  width: 250px;
  height: 100px;
  overflow: auto; /* Scrollbar muncul otomatis saat dibutuhkan */
}

2. Float

Float awalnya digunakan untuk membungkus teks di sekitar gambar (seperti tata letak koran). Elemen akan "mengapung" ke kiri atau kanan.

Kasus Nyata: Gambar dalam Artikel

IMAGE

Teks ini akan membungkus elemen di sampingnya karena elemen kotak ungu menggunakan float: left. Ini adalah cara klasik untuk membuat layout artikel yang menarik.

Istilah Baru: Clearfix. Saat menggunakan float, elemen induk seringkali "kehilangan" tingginya. Kita wajib menggunakan teknik clear: both atau Clearfix agar elemen di bawahnya tidak ikut berantakan. 
img {
  float: right;
  margin-left: 20px;
}
/* Menghentikan efek mengapung */
.footer { clear: both; }

3. Inline-block

Inline-block adalah jalan tengah. Ia membiarkan elemen berjejer ke samping (seperti inline), tetapi tetap bisa diatur lebar dan tingginya (seperti block).

Perbandingan:

  • inline: Bisa ke samping, tapi tidak bisa diberi width/height.
  • block: Bisa diberi width/height, tapi memaksa baris baru.
  • inline-block: Bisa ke samping DAN bisa diberi width/height.

Kasus Nyata: Kartu Produk atau Menu

.nav-item {
  display: inline-block;
  width: 100px;
  height: 40px;
  text-align: center;
}
Kelemahan: inline-block secara otomatis memberikan sedikit spasi (sekitar 4px) antar elemen. Hal ini karena browser menganggap spasi di kode HTML sebagai spasi nyata di layar.
