SASS Pre-processor Mastery

Apa itu SASS?

SASS adalah bahasa pemroses (pre-processor) yang akan diubah menjadi CSS biasa agar bisa dibaca browser.

Istilah Baru: Compilation (Kompilasi)
Browser tidak bisa membaca file .scss secara langsung. Kita membutuhkan alat (seperti ekstensi VS Code "Live Sass Compiler") untuk mengubahnya menjadi file .css.

Kenapa Pakai SASS?

  • Menulis kode jauh lebih cepat.
  • Struktur kode lebih rapi dan mudah dibaca.
  • Sangat efisien untuk proyek website skala besar.

Nesting (Bersarang)

Nesting memungkinkan kita menulis selector di dalam selector lain, mengikuti struktur HTML kita.

Perbandingan Kode:

/* DI SASS (.scss) */
nav {
  background: white;
  ul {
    margin: 0;
    li { display: inline-block; }
  }
}

/* HASILNYA DI CSS BIASA */
nav { background: white; }
nav ul { margin: 0; }
nav ul li { display: inline-block; }

Kasus Nyata: Navbar Complex

Saat membuat Navbar, Anda tidak perlu menulis .navbar .menu .item .link berulang kali. Cukup masukkan selector di dalamnya, sehingga kode Anda terlihat persis seperti struktur HTML.

SASS Variables

Sama seperti CSS Variables, tapi SASS Variables menggunakan simbol $. Bedanya, variabel SASS diproses sebelum sampai ke browser, sehingga lebih ringan.

Penulisan:

$warna-utama: #cf649a;
$jarak-standar: 20px;

.tombol {
    background-color: $warna-utama;
    padding: $jarak-standar;
}

Kasus Nyata: Branding Warna

Jika klien ingin mengubah warna brand dari Pink ke Biru di seluruh website, Anda hanya perlu mengubah satu baris di variabel $warna-utama saja.

Mixins (@mixin & @include)

Mixins adalah fitur paling sakti di SASS. Ini memungkinkan Anda membuat "template" grup kode CSS dan menggunakannya kembali.

Contoh Penggunaan:

/* Membuat Template */
@mixin pusat-elemen {
    display: flex;
    justify-content: center;
    align-items: center;
}

/* Menggunakan Template */
.hero-content {
    @include pusat-elemen;
    height: 100vh;
}

Kasus Nyata: Centering & Flexbox

Cukup buat satu mixin dan panggil dengan @include. Ini mengurangi duplikasi kode secara drastis!

Panduan Penggunaan SCSS

Berikut adalah panduan runtut penggunaan SCSS, dari instalasi hingga fitur-fitur utamanya.

1. Tahap Persiapan: Instalasi Compiler

Cara termudah untuk pemula adalah menggunakan ekstensi VS Code:

  • Buka VS Code.
  • Cari ekstensi bernama "Live Sass Compiler" (oleh Glenn Marks).
  • Klik Install.
  • Setelah terinstal, klik tombol "Watch Sass" di Status Bar bawah VS Code.

2. Struktur File dan Folder

Gunakan Partials (file dengan awalan underscore) agar kode rapi:

  • main.scss (File utama yang meng-import semua file).
  • _variables.scss (Penyimpanan warna dan font).
  • _buttons.scss (Gaya khusus tombol).

3. Menulis Kode SCSS

A. Variables

$warna-utama: #00b894;
$warna-gelap: #2d3436;

body {
    background-color: $warna-utama;
    color: $warna-gelap;
}

B. Nesting

.nav {
    background: white;
    ul {
        margin: 0;
        li {
            display: inline-block;
            a {
                color: blue;
                &:hover { color: red; }
            }
        }
    }
}

C. Mixins

@mixin pusat-elemen {
    display: flex;
    justify-content: center;
    align-items: center;
}

.box { @include pusat-elemen; }

D. Import

@import 'variables';
@import 'buttons';
@import 'layout';

4. Proses Kompilasi

Setiap kali Anda menekan Ctrl + S pada file .scss, compiler otomatis membuat file .css.

Penting: Di HTML, tetap panggil file CSS hasil kompilasi:
<link rel="stylesheet" href="main.css">

5. Keuntungan SCSS

  • Maintenance Cepat: Ubah 1 variabel untuk merubah seluruh web.
  • Keterbacaan: Struktur rapi, tidak ada selector panjang yang membingungkan.
  • Logika: Bisa operasi matematika dan fungsi warna (darken, lighten).
