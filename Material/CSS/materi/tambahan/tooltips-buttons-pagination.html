UI Components Mastery

Tooltips

Tooltips adalah label mengambang yang memberikan informasi singkat saat elemen diarahkan. Ini kunci untuk menjaga desain tetap bersih (Clean UI).

Kode Dasar:

.tooltip-box {
    position: absolute;
    bottom: 125%; /* Berada di atas trigger */
    visibility: hidden;
    opacity: 0;
}
.trigger:hover .tooltip-box {
    visibility: visible;
    opacity: 1;
}

Kasus Nyata: Toolbar Ikon Tanpa Teks

Pernahkah Anda menggunakan dashboard seperti YouTube atau Facebook? Banyak tombol hanya berupa ikon (seperti ikon "Sampah"). Tanpa tooltip, pengguna baru mungkin bingung apakah itu "Hapus Permanen" atau "Pindahkan ke Bin".

Buttons Interaction

Tombol bukan sekadar kotak berwarna. Ia harus memberikan *feedback* visual yang jelas saat disentuh atau diklik.

Prinsip Hover & Transition:

.btn {
    transition: all 0.3s ease; /* Transisi halus */
}
.btn:hover {
    transform: translateY(-2px); /* Efek melayang */
    filter: brightness(110%);   /* Sedikit lebih terang */
}

Kasus Nyata: Call to Action (CTA) Form

Dalam form pendaftaran, tombol "Simpan" harus terlihat lebih dominan daripada tombol "Batal". Ini membantu menuntun psikologi pengguna untuk menyelesaikan pendaftaran.

Pagination Navigation

Memecah data besar menjadi halaman-halaman kecil. Penting untuk kecepatan *loading* website.

Struktur Flexbox:

.pagination {
    display: flex;
    justify-content: center;
    gap: 8px;
}
.page-item.active {
    background-color: #0984e3;
    pointer-events: none; /* Halaman aktif tidak bisa diklik lagi */
}

Kasus Nyata: E-Commerce Product Listing

Saat Anda mencari "Sepatu" di Tokopedia, ada ribuan produk. Pagination memungkinkan browser hanya memuat 20-40 produk per halaman, sehingga HP pengguna tidak nge-lag saat membukanya.
